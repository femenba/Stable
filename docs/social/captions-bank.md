## CTA Strategy

### CTA Hierarchy (strongest to softest)
1. **Save this** — best for carousels and education content (saves = algorithmic gold)
2. **Send this to someone** — best for validation posts (reach multiplier)
3. **Drop it below** — best for question-based posts (comments boost reach)
4. **Try it free** — product CTAs, always with link framing
5. **Link in bio** — keep this for posts where the app is explicitly the answer

### CTA Rules
- One CTA per post. Never two.
- Never "like and follow" — it reads as desperate and is penalised
- Never "smash the notification bell" — wrong brand voice
- Always phrase CTAs as invitations, not commands

### CTA Phrasing Bank
```
→ save this for the next time it happens
→ send this to the person who needs to hear it
→ what's your experience? drop it below
→ try it free — link in bio
→ which one hits hardest? vote below
→ share this with someone who'd say "this is so me"
→ reply with your version of this
→ does this land for you?
→ what would you add?
→ follow for more of this
```
